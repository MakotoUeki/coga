Objective 1: Help users understand what things are and how to use them

User Needs Patterns Scenarios
Clear Purpose
  • Make the purpose of your page clear
  • Make the purpose of each section clear
  • Make it easy to find the most important things on the page
  • Group and highlight information with graphical indicators
  • Ensure the hierarchy of the site and menu structure is logical, easy to identify and navigate
  • Use symbols that help the user
  • Frank: Making a doctors appointment
  • Maria: Pressing the correct button
  • Maria: Dynamic website elements make key website information difficult to locate
  • Tom: Understanding where information is in a hierarchical structure
  • Carolyn: Focus
Clear Operation
  • Make it clear what are controls and how they should be used
  • Ensure controls and content do not move unexpectedly
  • Make the relationship clear between controls and what they affect
  • Use a design that the user is likely to recognize and understand
  • Make it easy to find the most important things on the page
  • Use a consistent visual design
  • Clearly state the results and disadvantages of actions, options, and selections
  • Alison: Learning how to use new technologies and interfaces.
  • Frank: Coping with icons that are not recognisable.
  • Frank: Using the heating.
  • Anna: Finding important information.
  • Gourge: Controls on videos and popup windows.
  • Amy: Coping with poor layouts and illogical navigation.
  • Sam: Using edit boxes where the instructions disappear.
  • Sam: Trying to activate elements that have been mis-recognised.
